# Support

Need help with Churn Guard? Here's where to go.

## Start here

- 📖 **[Wiki](https://github.com/PJH720/churn-guard/wiki)** — project background, the 4-notebook pipeline, data conventions, and milestone schedule.
- 🏗️ **[Architecture](wiki/Architecture.md)** & **[Roadmap](wiki/Roadmap.md)** — how the pipeline fits together and what's planned.
- 🗺️ **[Project Board](docs/PROJECT_BOARD.md)** & **[milestones](docs/milestones/)** — current work and Definition of Done per phase.

## Asking questions

- 💬 **[Discussions](https://github.com/PJH720/churn-guard/discussions)** — open questions on methodology, metric choice (Recall/F1/ROC-AUC), data handling, or retention ideas. No question is too small (새싹반 friendly!).
- 🐛 **[Issues](https://github.com/PJH720/churn-guard/issues/new/choose)** — pick a structured form: Bug, Data Quality, EDA/Analysis, Modeling Experiment, Retention Idea, or Documentation.

## For 새싹반 teammates

Coordinate through the class thread and your assigned mentor for project-level questions, scheduling, and Midterm (6/26) / Demo Day (7/10) prep.

## Common first stop

Notebook won't run? It's almost always the hardcoded Kaggle `file_path` — point it at the repo-root CSV. See [CONTRIBUTING.md](CONTRIBUTING.md#local-setup).

## What this is not

This is a 2-week class project, not a supported product. There is no SLA — responses are best-effort.
